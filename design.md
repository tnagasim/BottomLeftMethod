# bottom_left2.py 設計書

## 1. 概要

**ボトムレフト法 (Bottom-Left Method)** による多次元矩形パッキングアルゴリズムの実装。

ボトムレフト法とは、矩形を収納箱（ケース）に詰める際に、可能な限り「左下（低次元方向）」に配置しようとするヒューリスティックアルゴリズムである。本実装は2次元だけでなく、3次元以上にも対応している。

---

## 2. クラス構成

```
Rect          矩形（配置候補・配置済み矩形を表す）
Case          収納箱（ケース）を表す
BottomLeftMethod  ボトムレフト法の配置アルゴリズム
```

---

## 3. データ構造

### 座標表現

矩形はすべて **minmax_p** という `shape=(2, N)` の NumPy 配列で表現される。

|   インデックス   |           意味           |
| ---------------- | ------------------------ |
| `minmax_p[0, :]` | 各次元の最小座標（始点） |
| `minmax_p[1, :]` | 各次元の最大座標（終点） |

例（2次元の場合）:
```
minmax_p = [[x_min, y_min],
            [x_max, y_max]]
```

### 配置済み矩形リスト（stacked）

`Case.stacked` は `shape=(K, 2, N)` の NumPy 配列。K は配置済み矩形数、N は次元数。

---

## 4. クラス詳細

### 4.1 `Rect`

矩形1つを表すクラス。候補点の生成・検証に使用する。

|         メソッド          |                       説明                       |
| ------------------------- | ------------------------------------------------ |
| `get_size()`              | 矩形のサイズを返す（`max_p - min_p`）            |
| `is_feasible_point(case)` | ケース内の配置済み矩形と重ならないか判定         |
| `is_inside_of(case)`      | ケースの境界内に収まるか判定                     |
| `create(size)`            | サイズリストからRectを生成するファクトリメソッド |

**`is_feasible_point` の判定ロジック:**

各次元において、対象矩形と配置済み矩形の重複区間を計算する。
```
重複区間 = [max(min_p同士), min(max_p同士)]
```
重複区間が存在しない（`min >= max`）次元が1つでもあれば、干渉なし（配置可能）と判定する。

---

### 4.2 `Case`

収納箱を表すクラス。箱のサイズと配置済み矩形を管理する。

|          メソッド          |                       説明                       |
| -------------------------- | ------------------------------------------------ |
| `stack(rect_size, method)` | 矩形を配置し `stacked` に追加する                |
| `create(size)`             | サイズリストからCaseを生成するファクトリメソッド |

`create(size)` 時、`stacked` は `shape=(0, 2, N)` の空配列で初期化される。

---

### 4.3 `BottomLeftMethod`

ボトムレフト法のアルゴリズム本体。

|           メソッド           |                           説明                            |
| ---------------------------- | --------------------------------------------------------- |
| `calc_rect_point(rect_size)` | 矩形の配置位置を決定するメインメソッド                    |
| `make_candidates(rect_size)` | 配置候補点リストを生成する                                |
| `make_cand0()`               | 候補タイプ0：原点 `[0, 0, ..., 0]`                        |
| `make_cand1()`               | 候補タイプ1：配置済み矩形の各辺に隣接した位置             |
| `make_cand2()`               | 候補タイプ2：配置済み矩形の辺の交差位置（順列組み合わせ） |

---

## 5. アルゴリズム詳細

### 5.1 メインフロー（`calc_rect_point`）

```
1. make_candidates(rect_size) で全候補点を生成
2. ケース内に収まる候補を絞り込む（is_inside_of）
3. 配置済み矩形と干渉しない候補を絞り込む（is_feasible_point）
4. ボトムレフト基準で最適な位置を選択
```

### 5.2 候補点の生成

配置済み矩形が K 個、次元数が N の場合：

| 種別  | 生成数 |                      内容                      |
| ----- | ------ | ---------------------------------------------- |
| cand0 | 1      | 原点（常に候補）                               |
| cand1 | K × N  | 配置済み各矩形の各次元の最大辺を起点とした位置 |
| cand2 | N! × N | 配置済み矩形の順列から生成される交差位置       |

**cand1 の生成:**

各配置済み矩形 `r` と各次元 `i` に対して、次元 `i` の最小座標を `r[1, i]`（r の最大辺）にし、他次元は 0 とした候補を生成する。

**cand2 の生成:**

配置済み矩形 K 個から N 個を選んだ順列 `(r_0, r_1, ..., r_{N-1})` に対して、次元 `i` の最小座標を `r_i[1, i]` とした候補を生成する。これにより、複数の矩形の辺が交差するコーナー位置を網羅する。

### 5.3 ボトムレフト選択基準

```python
min(blfp, key=lambda v: tuple(v.minmax_p[0, ::-1]))
```

`minmax_p[0, ::-1]` は最小座標を次元の逆順（最終次元から優先）でソートする。

2次元の場合 `[x, y]` → `[y, x]` となり、**y（縦）を優先して小さい位置**、次に **x（横）が小さい位置** を選ぶ。これが「ボトムレフト（下・左優先）」の選択基準となる。

---

## 6. 使用例

```python
# 10x10 のケースに矩形を詰める
case = Case.create([10, 10])
method = BottomLeftMethod(case)

# 矩形を順番に配置
case.stack([3, 4], method)  # 3x4 の矩形を配置
case.stack([5, 2], method)  # 5x2 の矩形を配置
```

## 8. 制約・注意事項

- 配置済み矩形が0個の場合、`make_cand1` / `make_cand2` は空の配列を返す。`is_feasible_point` も常に `True` を返す。
- `make_cand2` は `itertools.permutations(stacked, N)` を使用するため、配置済み矩形数 K が増えると候補数が急増する（計算量 O(K! / (K-N)!) ）。
- 配置可能な位置が存在しない場合、`min(blfp, ...)` で例外が発生する（空リストへの `min` 呼び出し）。
